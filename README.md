RAG System for Superstore Sales Dataset
A Retrieval-Augmented Generation (RAG) system that answers natural language questions about US retail sales data from 2014–2017 using a local LLM.
Requirements

Python 3.10+
Ollama installed and running locally

Installation
1. Clone the repository and navigate to the project folder
bashcd rag
2. Install Python dependencies
bashpip install -r requirements.txt
3. Download the dataset
Download the Superstore Sales dataset from Kaggle:
https://www.kaggle.com/datasets/vivek468/superstore-dataset-final
Place the file Sample - Superstore.csv in the project root folder.
4. Install and start Ollama
bash# Download Ollama from https://ollama.com then run:
ollama pull phi3
ollama serve
Usage
First run — builds the database and runs all queries:
bashpython main.py
This will:

Load and prepare the dataset
Create 10,027 text documents
Embed and store them in ChromaDB
Run 5 example queries using the RAG pipeline

Subsequent runs — skip rebuilding the database:
Set rebuild = False in the main() function to load the existing ChromaDB collection instead of rebuilding it from scratch. This makes subsequent runs much faster.
python# In main():
rebuild = False  # change this line
```

## How It Works

**Indexing (runs once)**
1. The CSV is loaded and each row is converted into a descriptive sentence
2. Aggregated summaries are generated by region, category, year, and region+year
3. Statistical summaries are computed for profit margins, top/bottom performers, and discounts
4. All 10,027 documents are embedded using `all-MiniLM-L6-v2` and stored in ChromaDB with metadata

**Retrieval (runs per query)**
1. The query is embedded using the same MiniLM model
2. ChromaDB performs similarity search optionally filtered by metadata fields (region, category, year, chunk_type)
3. Top-k most relevant chunks are returned

**Generation (runs per query)**
1. Retrieved chunks are formatted into a context block
2. A prompt is built with a system role, retrieved context, rules, and the question
3. Phi-3 Mini via Ollama generates a grounded answer using one of three strategies:
   - Zero-shot — answer directly from context
   - Few-shot — follow an example answer format
   - Chain-of-thought — reason step by step before concluding

## Example Queries

| Query | Strategy | Filter |
|---|---|---|
| What were the total sales each year? | Zero-shot | chunk_type: yearly_summary |
| Which category had the highest profit? | Few-shot | chunk_type: category_summary |
| How did the West region perform? | Chain-of-thought | chunk_type: regional_summary |
| Compare regions in 2017 | Chain-of-thought | year: 2017 |
| Most discounted sub-categories? | Zero-shot | chunk_type: statistical_summary |
```
